### MP4 Transcoding (On-Demand)
- Click MP4 in Browse Files ‚Üí `POST /transcode {path}` ‚Üí ffmpeg pulls camera MP4 over HTTP, transcodes to 480p HLS
- Output: `/data/data/com.termux/files/home/.transcode/<session_id>/trans.m3u8`
- Served via `/transcode/<session_id>/trans.m3u8`
- Sessions auto-expire after 10 min idle; ffmpeg reads with `-re` (realtime), first segment in ~5s
- Transcode params: `scale=640:-2`, `libx264 ultrafast`, `crf 28`, `an`, `hls_time 3`, `hls_list_size 30`
- Bug: `delete_out_of_range` is NOT a valid ffmpeg hls_flag ‚Äî removed it (was causing ffmpeg to fail silently)
- Bug: Python 3.13 SIGCHLD handler prevents zombie reaping ‚Üí `signal.signal(signal.SIGCHLD, signal.SIG_DFL)` at top of server
- Bug: camera rejects HTTP requests without `User-Agent` ‚Üí added `-user_agent "Mozilla/5.0..."` to ffmpeg
- Bug: transcode dir empty (ffmpeg defunct) ‚Äî fixed by: valid hls_flags, stderr to log file (not PIPE), SIGCHLD reset

### Proxy Bug (2026-04-16 ~09:30)

- Fixed: replaced `return` with `self._proxy_camera("/" + p[4:].lstrip("/"))`
