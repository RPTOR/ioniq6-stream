## SIGCHLD Fix — Critical (10:02 UTC)
- **Problem**: Duplicate ffmpeg processes kept spawning every 10s watchdog cycle.
- **Root cause**: `signal.signal(signal.SIGCHLD, signal.SIG_DFL)` — kernel was auto-reaping dead ffmpeg children before `poll()` could detect them, causing the watchdog to think ffmpeg was dead and respawn it.
- **Fix**: Changed to `signal.signal(signal.SIGCHLD, signal.SIG_IGN)` in `stream_server.py` on the PHONE (via SSH).
- **Also**: Changed `HTTPServer` → `ThreadingHTTPServer` in `stream_server.py` (phone) to handle concurrent requests.
- **Verified**: Only 1 ffmpeg process running now; segments writing cleanly; no more duplicate spawns.
